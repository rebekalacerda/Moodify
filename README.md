## 📌 Descrição
Projeto desenvolvido para demonstrar o uso do fluxo de versionamento com Git e GitHub.

## 🔄 Fluxo Git Utilizado
- Criação de branch (feature-melhorias)
- Realização de commits
- Envio para o repositório (push)
- Criação de Pull Request
- Merge com a branch principal (main)

## 📝 Mini-Relatório de Decisões Técnicas
Durante o desenvolvimento, optamos por utilizar branches separadas para organizar as alterações e evitar conflitos no código principal.

Os commits foram realizados com mensagens descritivas para facilitar o entendimento das mudanças.

O uso de Pull Requests permitiu revisar e integrar as alterações de forma controlada, garantindo maior qualidade no desenvolvimento.
